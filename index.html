    Three main diffrences between web1.0 web2.0 and web3.0

    Differences in user interaction:
    web1.0 is considered read only, meaning that there is no real interaction between
    advertisers and readers.
    On the other hand, a web-2.0 type of read-write enables interaction between different users
    by means of tagging, content editing and many other means.
    In web3.0 by user sharing and development in the field of artificial intelligence, we can learn
    the user preferences and make recommendations and comfortable kissing systems suitable
    for the current user.

    Differences at Web page update frequency (static vs. dynamic updates):
    In web1.0, static updates are made mostly by the webmaster.
    On the other hand, web2.0 and web3.0 users can update what
    usually causes more dynamic updates (more frequent).

    Differences in internet search capabilities and results:
    In web1.0, search results will usually yield macro content that is not necessarily very accurate,
    but in web2.0 you can search for more successful Google search engines that return results with
    micro content, but the content we receive will not necessarily match what the user searched for.
    In web3.0 machine learning and semantic search can be used and thus the chances that the
    results obtained will meet the need of the user will greatly increase.

    Example of use case: ask the web about "games for PC":


    web1.0 Will return as a result several static pages in the field of computer games,
    some relevant and some of them are old and may not match my operating system type even.
    web2.0 Will return pages containing computer games,
    probably newer games, ie newer and more familiar games.
    web3.0 Will probably return pages containing games so that the
    results will suit me personally. If I've been looking for action
    games or shooting games in the past, I'll probably return pages
    that contain newer and more familiar games, whose genre is action / shooting games.
